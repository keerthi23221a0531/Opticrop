# Problem-Solution Fit Canvas

**Project Name:** OptiCrop — Smart Agricultural Production Optimization Engine
**Team ID:** _xxxxxx_
**Date:** _add date_

| Block | Content |
|---|---|
| **1. Customer Segment(s)** | Small-to-medium landholding farmers; agri-cooperatives; agricultural researchers/policymakers |
| **2. Customer Jobs-to-be-Done** | Decide which crop to plant given current soil & weather conditions |
| **3. Triggers** | Start of a new planting season; poor yield in a previous season; access to a smartphone/PC |
| **4. Emotions** | Anxiety about financial risk; desire for confidence in the decision |
| **5. Available Solutions (existing alternatives)** | Local agri-extension advice, trial-and-error, generic government advisories |
| **6. Customer Limitations** | Limited technical/agronomic expertise; limited access to soil-testing labs; low tolerance for complex tools |
| **7. Problem Root Cause** | No accessible, data-driven, personalized way to map soil/climate parameters to the best-fit crop |
| **8. Behaviour** | Farmers rely on habit/experience rather than data when choosing crops |
| **9. Proposed Solution** | OptiCrop: an ML-powered web app that takes N, P, K, temperature, humidity, pH, rainfall and returns the recommended crop instantly |

## Fit Statement
Because farmers (**customer segment**) need to choose the right crop for their land (**job**) but lack a simple, data-backed way to do so (**root cause/limitation**), OptiCrop provides an ML-driven recommendation through an easy web form (**solution**), directly closing that gap.
